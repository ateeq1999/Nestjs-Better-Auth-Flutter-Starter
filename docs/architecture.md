# Architecture

Feature-first, four-layer: **Provider → Repository → Cubit/BLoC → View**. Dependencies flow down; events flow up.

```
View            (widgets, forms, navigation)
  │ context.read<Cubit>()
  ▼
Cubit / BLoC    (UI state machine; emits sealed states)
  │ calls
  ▼
Repository      (business logic; throws ApiException; unwraps envelopes)
  │ calls
  ▼
Provider        (raw Dio calls to /v1/api/...; returns Response<dynamic>)
  │
  ▼
DioService      (Bearer token + 401 refresh interceptor)
```

## Why four layers?

- **Provider** knows about URLs + Dio only. Mock-friendly.
- **Repository** owns error mapping (`DioException` → `ApiException`) and envelope unwrapping ([response_parser.dart](../lib/app/core/utils/response_parser.dart)). Swap Dio for another HTTP client by rewriting one layer.
- **Cubit/BLoC** owns UI state transitions. Never imports Dio.
- **View** owns widgets + navigation side effects. Never imports repositories directly — it reads them via `context.read<Repo>()` inside a Cubit constructor.

## Dependency injection

No service locator. Everything is injected through the `flutter_bloc` provider tree in [main.dart:187-215](../lib/main.dart#L187-L215):

```dart
MultiRepositoryProvider(
  providers: [
    RepositoryProvider<AuthService>.value(value: authService),
    RepositoryProvider<AuthRepository>.value(value: authRepository),
    RepositoryProvider<UserRepository>.value(value: userRepository),
    RepositoryProvider<AdminRepository>.value(value: adminRepository),
    RepositoryProvider<OrganizationRepository>.value(value: orgRepository),
    RepositoryProvider<FeatureFlags>.value(value: featureFlags),
  ],
  child: MultiBlocProvider(
    providers: [
      BlocProvider<AuthBloc>.value(value: _authBloc),
      BlocProvider<ThemeCubit>(create: (_) => ThemeCubit(prefs)),
    ],
    child: MaterialApp.router(routerConfig: _routerHolder.router),
  ),
)
```

Per-screen cubits are created inside their `GoRoute` builder with `context.read<Repo>()` — see [app_router.dart:100-106](../lib/app/core/router/app_router.dart#L100-L106).

## Auth state & routing

[AuthBloc](../lib/app/modules/auth/auth_bloc.dart) is the single source of truth for who is signed in. It emits a sealed state:

- `AuthInitial` / `AuthLoading` — hydrating, stay on splash
- `AuthAuthenticated(user, token)` — home + protected routes
- `AuthUnauthenticated` — sign-in + guest routes

The router listens to the bloc stream via `_BlocChangeNotifier` ([app_router.dart:309](../lib/app/core/router/app_router.dart#L309)) and re-evaluates its `redirect` on every emission. This is how sign-out drops you back to `/sign-in` without any explicit navigation — the bloc emits `AuthUnauthenticated` and the router redirects.

Admin routes additionally check `authState.user.role == 'admin'` ([app_router.dart:83-87](../lib/app/core/router/app_router.dart#L83-L87)).

## Response envelope

Most endpoints return:

```json
{ "success": true, "data": {...}, "meta": { "timestamp": "...", "requestId": "..." } }
```

Repositories call `unwrapEnvelope(response.data)` to strip the wrapper. Auth endpoints (`/v1/api/auth/*`) return the payload directly — no unwrap needed.

## Errors

`DioException` is caught at the repository boundary and converted to `ApiException` ([app_exception.dart](../lib/app/core/errors/app_exception.dart)):

- `statusCode`, `message`, `code`
- `fieldErrors: Map<String, String>?` — per-field validation errors, consumed via `TextFormField.forceErrorText` in forms
- `details: List<String>?` — free-text error messages

Cubits catch `ApiException` and emit a `*Failure` state; views show a snackbar or inline field errors.

## Pagination

Paginated endpoints (`AdminRepository.listUsers`, `listAuditLogs`) read cursor state from `meta.pagination`:

```json
"meta": { "pagination": { "limit": 20, "hasNextPage": true, "nextCursor": "abc" } }
```

See the `_pagination` helper in [admin.repository.dart](../lib/app/data/repositories/admin.repository.dart).

## CancelToken

List cubits that can be left mid-flight (admin users, audit logs, org list) hold a `CancelToken?`. On a new `loadX()` call they cancel the previous token and replace it; on `close()` they cancel outstanding requests. See [org_list_cubit.dart](../lib/app/modules/organizations/org_list_cubit.dart) for the minimal pattern.

## Persistence

| Data | Store | Location |
|---|---|---|
| Bearer token + user JSON | `flutter_secure_storage` | [auth_service.dart](../lib/app/services/auth_service.dart) |
| Theme preferences | `SharedPreferences` | [theme_cubit.dart](../lib/app/modules/theme/theme_cubit.dart) |
| HydratedBloc state | Application docs dir | initialised in [main.dart:41](../lib/main.dart#L41) |

Never put tokens in `SharedPreferences` — it's not encrypted.
